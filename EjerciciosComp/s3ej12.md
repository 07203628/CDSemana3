### Ejercicio 12: El Proceso de Data Science

## 1. ¿Qué es el ciclo CRISP-DM?
CRISP-DM (Cross-Industry Standard Process for Data Mining) es una metodología estándar para desarrollar proyectos de analítica y minería de datos. Su objetivo es guiar el trabajo desde la comprensión del problema de negocio hasta la implementación de una solución útil.

Se usa porque:
- ordena el proyecto en etapas claras,
- reduce errores por falta de contexto de negocio,
- permite iterar cuando los resultados no cumplen lo esperado.

## 2. ¿Cuáles son las fases del proceso de ciencia de datos?
Aunque pueden variar según la organización, normalmente se consideran estas fases:

1. **Definición del problema**
Se identifica qué decisión se quiere mejorar y qué métrica indicará éxito.

2. **Recolección de datos**
Se obtienen datos desde bases, APIs, archivos o sistemas transaccionales.

3. **Limpieza y preparación**
Se corrigen valores faltantes, formatos, duplicados y se transforman variables.

4. **Análisis exploratorio (EDA)**
Se estudian distribuciones, relaciones, outliers y patrones iniciales.

5. **Modelado**
Se entrenan modelos estadísticos o de machine learning para resolver el problema.

6. **Evaluación**
Se comparan resultados con métricas técnicas y objetivos de negocio.

7. **Despliegue e implementación**
La solución se integra en un entorno real (dashboard, API, reporte automatizado, etc.).

8. **Monitoreo y mejora continua**
Se revisa el desempeño en producción para recalibrar o reentrenar cuando sea necesario.

Relacionándolo con CRISP-DM, las seis fases clásicas son:
- Comprensión del negocio,
- Comprensión de los datos,
- Preparación de los datos,
- Modelado,
- Evaluación,
- Despliegue.

## 3. ¿Qué es el MVP (Minimum Viable Product) en ciencia de datos?
Un MVP en ciencia de datos es la versión mínima funcional de una solución analítica que ya entrega valor real, aunque no esté completamente optimizada.

Características de un MVP de Data Science:
- resuelve un caso de uso específico,
- usa el menor esfuerzo y tiempo posible,
- permite validar si la solución realmente ayuda al negocio,
- se construye para aprender rápido y mejorar con evidencia.

**Ejemplo:**
En lugar de construir un sistema complejo de predicción de abandono desde el inicio, se crea primero un modelo simple con pocas variables y un reporte semanal para el equipo comercial. Si genera valor, se escala a una versión más robusta.

## Conclusión
CRISP-DM aporta una guía metodológica; el proceso de ciencia de datos describe el flujo completo de trabajo; y el MVP permite validar valor de forma temprana antes de invertir en soluciones complejas.
